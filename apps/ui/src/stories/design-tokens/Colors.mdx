# After6ix Enhanced Color System

The After6ix design system is inspired by twilight hours when developers do their best work. Our enhanced color tokens provide a comprehensive foundation for building beautiful, accessible interfaces that work seamlessly in both light and dark modes.

## Design Philosophy

Our color system follows David Ogilvy's principles of truth and helpfulness, providing:
- **Truthful naming** - Each color tells its story (Twilight Purple, Deep Sky, Electric Cyan)
- **Helpful accessibility** - WCAG AA compliant contrast ratios across all combinations
- **Factual usage guidance** - Clear semantic meanings for every color variation
- **Consistent brand expression** - Unified twilight-themed palette that scales

## Accessibility Standards

All color combinations meet or exceed WCAG 2.1 standards:
- **AAA (7:1)**: Primary text combinations
- **AA (4.5:1)**: Secondary text and UI elements
- **Enhanced contrast**: Dark mode optimizations for OLED displays

## Usage Guidelines

### TypeScript Integration
```tsx
import { colors } from '@after6ix/ui/tokens';

// Brand colors - use numeric scale
const primaryButton = {
  backgroundColor: colors.primary[600],      // Base brand color
  color: colors.text.light.inverse,         // Proper contrast
  border: colors.border.light.focus,        // Focus ring
};

// Semantic colors - use named variants
const successMessage = {
  backgroundColor: colors.semantic.success[100], // Light background
  color: colors.semantic.success[700],           // Dark text
  borderColor: colors.semantic.success[500],     // Medium border
};
```

### CSS Custom Properties
```css
/* Use theme-aware surface colors */
.card {
  background-color: hsl(var(--surface-primary));
  color: hsl(var(--text-primary));
  border: 1px solid hsl(var(--border-primary));
}

/* Semantic color usage */
.error-state {
  background-color: hsl(var(--error-50));
  color: hsl(var(--error-700));
  border-color: hsl(var(--error-500));
}
```

### Best Practices

1. **Use semantic colors for meaning** - Success, error, warning, info
2. **Use theme-aware tokens** - Surface, text, and border variants
3. **Test in both themes** - Ensure accessibility in light and dark modes
4. **Follow contrast guidelines** - Use appropriate text/background pairings
5. **Maintain brand consistency** - Primary colors for brand elements
